Sintaxis

<form action="archivo.ej" method="get/post" enctype="multipart/form-data">
       <contenido del formulario>
</form>

action: Define la acción que se realiza cuando se envía un formulario. Especifica una dirección URL dentro del servidor donde enviar el formulario.

method: Define de que manera se van a transmitir los datos hacia el archivo que se define en el action.
Existen dos métodos:
- get (por defecto): cuando la información a enviar no es sensible; los datos del formulario son visible en la dirección url de la página. Recomendado para poca cantidad de texto.
- post: cuando la información es sensible; ofrece mayor seguridad dado que los datos no están visibles en la dirección de la página.

enctype: Especifica cómo los datos serán codificados el enviarlos al servidor. Sólo se puede aplicar si el método utilizado es post.
Las opciones son: application/x-www-form.urlencoded (por defecto), multipart/form-data y text/plain.

Tipos de inputs

Text

Este input permite el ingreso de datos alfanuméricos. Incluye caracteres especiales.

Number

Sirve para el ingreso de datos numericos. Puede establecerse un mínimo y un máximo; además, permite la validación de datos.

Radio

El usuario podrá seleccionar una sola opción del grupo de radios. Para identificarse como grupo de radios, deben tener el mismo name pero distinto value.
El atributo checked permite establecer una opción predefinida.

Acepto las condiciones

Checkbox

El usuario podrá seleccionar varias opciones del grupo de radios. Para identificarse como grupo de checkboxes, deben tener el mismo name pero distinto value.
El atributo checked permite establecer una opción predefinida.

¿Qué lenguajes sabés?

Password

Ideado para el ingreso de contraseñas ya que no pueden visualizarse los caracteres ingresados.

Email

Ideado paa el ingreso de correos electrónicos. Este input posee una validación para identificar si el ingreso es un correo electrónico o no.

Tel

Ideado para el ingreso de números telefónicos. En versión mobile, el teclado es numérico.

Date

Ideado para el ingreso de fechas. Al tocar el ícono a la derecha del input, se despliega un calendario. Puede agregarse en atributos la posibilidad de validar valores mínimos y máximos.
Variantes de este input son: time, datetime-local, month, week.

Color

El usuario puede seleccionar un color o ingresarlo manualmente en valor rgb.

URL

Este input valida las url ingresadas.

Search

Sirve como entrada para realizar una búsqueda dentro de la página.

File

Ideado para seleccionar y subir archivos.

Otros campos útiles

Textarea

El elemento textarea define un campo de entrada de varias líneas. Los atributos rows y cols especifican el número de líneas y ancho respectivamente.

Select y Option

El elemento select va a tener un name declarado, dentro de este, los options son todas las opciones que se nos van a desplegar para poder elegir, este elemento va a tener el atributo value definido que es el valor que se va a enviar atraves de este formulario.
El atributo selected permite establecer una opción predefinida.

Datalist

Se establece una lista predefinida de opciones, la cual no condiciona al usuario; ya que este puede ingresar datos que no se encuentran en la misma.

Atributos de los campos

Autofocus

El atributo autofocus permite que una vez cargada la página, el campo al que se le asigna sea el que tenga el cursor.

Placeholder

Este atributo será el texto alternativo dentro del input. Al ingresar el dato, éste desaparece.

Required

Incluir el atributo required genera que ese campo sea obligatorio.

Value

Lo que se establece en el atributo value, será el contenido por defecto.

Checked

El atributo checked permite establecer una opción predefinida. Se puede utilizar en inputs radio y checkbox.

Acepto las condiciones

Selected

Este atributo permite establecer una opción predefinida. Se puede utilizar en campo datalist.
